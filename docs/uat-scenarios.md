# QLink UAT 시나리오 (개발 서버 환경)

> 실제 사용자 인수 테스트(UAT) 수행용 체크 시트입니다. 각 시나리오를 직접 수행하며 **웹 와이드 / 웹 모바일 / 앱** 열에 결과 스크린샷을 붙여넣으세요.

## 테스트 환경

| 구분 | 대상 |
|---|---|
| **웹 와이드** | https://dev.qlinkapps.com (PC 넓은 화면 / 사이드바 레이아웃) |
| **웹 모바일** | https://dev.qlinkapps.com (좁은 화면 / 모바일 레이아웃) |
| **앱** | iOS Simulator에 빌드한 앱 |
| **서버** | https://dev.api.qlinkapps.com |

## 표 보는 법
- **웹 와이드 / 웹 모바일 / 앱**: 수행 후 스크린샷을 붙이는 칸. `N/A` = 해당 환경엔 그 기능이 없음(수행 불필요).
- **입력**: 시나리오 수행 시 넣는 값. 미리 채워진 값은 그대로 사용, 빈 라벨(예: `네이버 아이디:`)은 본인 계정/데이터를 채워 진행.
- **성공 기준**: 사용자 입장에서 "이렇게 되면 통과".
- 시뮬레이터 제약: 푸시 알림·카메라(QR)는 시뮬레이터에서 동작하지 않아 실기기에서 별도 확인이 필요합니다.

## 알려진 제약 (개발 환경 기준)
- **시뮬레이터 미지원**: 푸시 알림(AUTH-011, NOTI-004), QR 카메라(LINK-010) → 실기기에서 별도 확인.
- **웹 검색**: 실제 검색 결과가 아니라 "구글에서 검색" 새 탭 열기(임시).
- **AI 요약/키**: AI 제공자 설정 상태에 따라 동작이 달라질 수 있음.
- **타 앱에서 공유받기(공유 익스텐션)**: 아직 미구현.
