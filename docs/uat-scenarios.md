# QLink UAT 시나리오 (개발 서버 환경)

> 위에서 아래로 **한 계정으로 한 번에** 수행하도록 순서를 짰습니다(로그인 → 생성 → 수정 → 조회 → 삭제). 별도 테스트 데이터 준비 없이, 앞 단계에서 만든 데이터를 뒤 단계에서 그대로 조회/수정/삭제합니다.
> 각 행의 **웹 와이드 / 웹 모바일 / 앱** 칸에 수행 결과 스크린샷을 붙이세요.

## 테스트 환경

| 구분 | 대상 |
|---|---|
| **웹 와이드** | https://dev.qlinkapps.com (PC 넓은 화면) |
| **웹 모바일** | https://dev.qlinkapps.com (좁은 화면) |
| **앱** | iOS Simulator 빌드 앱 |
| **서버** | https://dev.api.qlinkapps.com |

## 수행 방법
- **진입**: No.1 개발자 로그인(또는 소셜 로그인 중 하나)으로 로그인한 뒤, 그 계정으로 끝까지 진행합니다.
- **소셜 로그인 검증**(No.2~5): 각 방식은 로그아웃 후 다시 로그인하며 따로 확인합니다.
- **칸 표기**: 빈 칸 = 스크린샷 붙이는 곳 / `N/A` = 그 환경엔 없는 기능.
- **입력**: 채워진 값은 그대로 사용, 빈 라벨(예: `네이버 아이디:`)은 본인 데이터로 채웁니다.
- **시뮬레이터 제약**: 푸시 알림·QR 카메라는 시뮬레이터에서 안 되며 실기기로 확인합니다.

## 알려진 제약 (개발 환경 기준)
- **시뮬레이터 미지원**: 푸시 알림(No.8, 42), QR 카메라 → 실기기에서 별도 확인.
- **웹 검색**: 실제 검색 결과가 아니라 "구글에서 검색" 새 탭 열기(임시).
- **AI 요약/키**: AI 제공자 설정 상태에 따라 동작이 달라질 수 있음.
- **타 앱에서 공유받기(공유 익스텐션)**: 아직 미구현.
- **회원 탈퇴(No.49)**: 비가역이므로 전체 시나리오의 가장 마지막, 별도 계정에서만 권장.
